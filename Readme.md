## Лабораторная работа 5 (2.3) - Автоматизация тестирования VK Видео

### Цель работы
Автоматизировать сценарии тестирования видео-хостинга VK Видео с использованием Selenium WebDriver на языке Java.

### Используемые технологии
- Java 21
- Gradle
- Selenium WebDriver 4.20.0
- TestNG
- WebDriverManager
- GitHub Actions (CI)

### Реализованные сценарии

#### Сценарий 1 (офлайн, с лайком)
Выполняется в видимом режиме браузера с использованием подготовленного профиля Chrome-vk-profile для авторизации.

**Демонстрация работы сценария 1:**

![Демонстрация сценария 1](HW5(2.3).gif)

**Шаги сценария:**
1. Открытие страницы VK Видео
2. Поиск видео по ключевому слову "Токсис возьми телефон"
3. Переход на первое видео в результатах поиска
4. Нажатие кнопки "Подписаться"
5. Нажатие кнопки "Нравится" (лайк)
6. Перезагрузка страницы
7. Отписка, снятие лайка и завершение (приведение к исходному состоянию)

#### Сценарий 2 (headless для CI)
Выполняется в headless-режиме без авторизации для запуска в CI.

**Шаги сценария:**
1. Открытие страницы VK Видео
2. Поиск видео по ключевому слову
3. Переход на первое видео
4. Сохранение скриншотов и HTML-страниц
5. Проверка загрузки видеоплеера

### CI (GitHub Actions)

Workflow настроен на:
- Ручной запуск (workflow_dispatch)
- Автоматический запуск при push в ветки main/master
- Сохранение артефактов: скриншоты и HTML-страницы

**Артефакты CI:**
- Скриншоты каждого этапа выполнения теста
- HTML-код страниц для отладки
